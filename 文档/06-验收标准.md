# 06-验收标准

本项目的验收分为两类：工程验收和视觉验收。

## 工程验收

必须满足：

- PNG 尺寸正确，默认 `1080 x 1440`。
- HTML 打开无 console error（控制台错误）。
- 页面导出无 page error（页面错误）。
- 文字没有明显溢出。
- 产物不是滚动网页截图。
- 输出目录包含参数或 manifest（清单）。
- 使用素材有来源和授权记录。

公开工具页还必须满足：

- `node scripts/validate-public-tools.mjs` 通过。
- 工具页不依赖外网资源。
- 工具页不包含真实账号、第三方知名 IP、私有素材路径。
- 工具页的关键按钮和输入框有稳定选择器，方便 Playwright（浏览器自动化）验证。
- 参数导出 JSON 可以重新导入，且能恢复主要文本和布局参数。

交互式模板 PNG 还必须满足：

- `render-template.mjs` 能从 JSON 配置生成 PNG。
- `validate-template.mjs` 返回 `pass: true`。
- PNG 为 `1080 x 1440`。
- 输出目录包含 `template-render-manifest.json`、`template-validation-report.json`、`process-and-params.md`。
- 输出 HTML 和 manifest 中没有真实账号、第三方 IP、私有素材路径。

## 视觉验收

每个模板或模块按 1-5 分评分：

- 视觉冲击力
- 标题可读性
- 平台适配
- 模块可复用
- 风格一致性
- 素材版权风险

建议判定：

- 平均 4 分以上：可进入 `stable` 候选。
- 平均 3 分以上：保持 `beta`。
- 低于 3 分：不进入模板库。

## 自动验证不能替代人工判断

脚本能检查尺寸、错误、溢出，但不能判断“好不好看”。

视觉质量必须保留人工验收。

## 每次生成的最低交付

- PNG 文件
- HTML 渲染源
- 参数记录
- 验证结果
- 如果使用外部素材，必须有素材来源记录
- 如果使用交互式编辑器，必须保存导出的 JSON 参数
- 如果使用模板渲染脚本，必须保存模板验证报告
