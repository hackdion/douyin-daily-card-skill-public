# 11-模板渲染链路说明

本文件说明交互式编辑器如何进入自动 PNG 生产链路。

## 目标

让用户在 `工具/交互式封面底图编辑器.html` 里调好的参数，可以被 agent 长期复用，而不是只停留在浏览器截图。

## 输入

输入是编辑器导出的 JSON，格式参考：

```text
skills/douyin-daily-card/examples/template-config-public-safe.json
```

核心字段：

- `mode`：`cover` 或 `base`
- `preset`：早晨、午间、夜间、周一到周日对应的预设 ID
- `role`：公开安全 CSS 角色 ID
- `text`：账号、ID、标题、日期、标签文案
- `layout`：每个文字元素的 X、Y、字号

## 渲染

```bash
node skills/douyin-daily-card/scripts/render-template.mjs skills/douyin-daily-card/examples/template-config-public-safe.json --output /tmp/daily-card-template-test
```

脚本会：

1. 打开公开安全交互式编辑器。
2. 把 JSON 参数导入编辑器。
3. 固定画布为 `1080 x 1440`。
4. 截取画布 PNG。
5. 写入 `template-render-manifest.json` 和 `process-and-params.md`。

## 验证

```bash
node skills/douyin-daily-card/scripts/validate-template.mjs /tmp/daily-card-template-test
```

验证项：

- PNG 是否存在。
- PNG 是否为 `1080 x 1440`。
- HTML 画布是否为 `1080 x 1440`。
- console error 是否为 `0`。
- page error 是否为 `0`。
- 是否存在过程记录。
- 是否残留真实账号、第三方 IP、私有素材路径。

## 当前边界

- 这是 `beta` 链路，适合公开安全模板和参数验证。
- 它复用编辑器 HTML，不维护第二套视觉实现。
- 暂不做联网素材下载。
- 暂不做多模板批量渲染。
- 暂不替代日报正文分页脚本。
