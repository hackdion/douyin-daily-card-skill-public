# 02-素材授权台账规则

所有素材都必须能回答三个问题：

1. 从哪里来？
2. 能不能公开使用？
3. 如果别人复用这个仓库，是否知道要不要署名？

## 台账字段

候选素材记录在 `数据/公开素材候选表.csv`。

正式入库素材记录在 `数据/已入库素材清单.json`。

必填字段：

- `id`：素材唯一标识。
- `名称`：中文名称。
- `分类`：杂志 / 漫画 / 游戏 / 星际科技 / 字体 / 图标 / 纹理。
- `来源`：素材网站或项目名。
- `来源链接`：原始页面链接，不只写首页。
- `授权`：例如 CC0、CC BY 3.0、MIT、OFL。
- `是否需要署名`：是 / 否 / 待确认。
- `署名文本`：需要署名时必须填写。
- `适合模块`：背景、标题、角色、信息条、内容区、特效。
- `公开风险`：低 / 中 / 高。
- `状态`：候选 / 已入库 / 放弃 / 待确认。

## 风险分级

低风险：

- CC0 / Public Domain。
- 授权页清楚。
- 不包含真人、品牌标识、知名 IP。

中风险：

- CC BY，需要署名。
- NASA 等特殊机构素材，需要遵守额外规则。
- 来源页授权清楚，但素材内容可能有品牌或人物。

高风险：

- 授权不明。
- 来自游戏截图、动漫角色、影视角色。
- 只写“free”但没有 license。
- 需要商用授权但没有购买证明。

## 禁止入库

- 无来源链接。
- 无授权说明。
- 明显侵权素材。
- 不能说明是否允许再分发的素材。

## 发布前检查

公开发布前必须检查：

- `公开素材候选表.csv` 中没有高风险素材被标为已入库。
- `已入库素材清单.json` 中每个素材都有来源和授权。
- 需要署名的素材在 README 或素材说明里保留署名。
- 没有真实账号隐私信息。
