# 05-Agent使用流程

本文件说明任意 agent 应该如何使用这个项目。

## 标准流程

1. 读取用户需求。
2. 判断输出类型：封面、底图、日报正文页，还是整套图文。
3. 查询 `数据/模板配方注册表.json`。
4. 只自动选择 `stable` 模板。
5. 如果用户要求实验风格，才允许使用 `beta` 模板。
6. 填入文本、日期、账号、ID 和素材。
7. 生成 HTML 渲染源。
8. 导出 PNG。
9. 运行验证。
10. 报告输出路径、参数和验证结果。

## 交互式调参流程

当用户说“我要自己移动元素”“字体大小想调一下”“先在网页上试效果”时：

1. 打开 `工具/交互式封面底图编辑器.html`。
2. 让用户选择封面或底图、预设、角色占位。
3. 用左侧输入框修改文案。
4. 用滑杆调整 X、Y、字号。
5. 导出 JSON 参数。
6. 后续生成 PNG 时，把这份 JSON 参数写入过程记录。

这个编辑器当前是 `beta`，适合视觉探索和参数确认；正式发布 PNG 仍需要经过脚本验证。

## 禁止行为

- 不要把固定图片产物做成滚动网页。
- 不要跳过验证。
- 不要使用授权不明素材。
- 不要把第三方 IP 当成公开演示素材。
- 不要自动选择 `beta` 模板。
- 不要把 `beta` 编辑器调出的参数直接当成最终交付，必须再走验证。
- 不要在用户未要求时重写正文事实。

## 当信息不足时

如果用户没有指定风格：

- 使用通用稳定模板。
- 不要重新发明一个新风格。

如果用户没有素材：

- 使用公开演示素材。
- 或使用剪影 / 几何 / CSS 图形。

如果用户指定私有素材：

- 不要把私有素材复制到公开仓库。
- 输出目录应放在任务工作区或私有项目目录。

## 输出时必须说明

- 使用了哪个模板。
- 为什么选择这个模板。
- 输出了哪些文件。
- 验证是否通过。
- 是否存在素材授权注意事项。
