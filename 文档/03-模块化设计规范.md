# 03-模块化设计规范

模块是这个项目的核心。模板不是直接写死的页面，而是由模块组合出来的配方。

## 模块分类

背景模块：

- 负责整体氛围。
- 例：科技网格、旧报纸、漫画网点、星空、雷达、杂志分栏。

标题模块：

- 负责主标题表达。
- 例：居中大标题、左侧大标题、斜切标题、爆炸贴纸标题、杂志刊头标题。

角色模块：

- 负责视觉焦点。
- 例：右下角角色、半透明水印、剪影背光、破框角色。

信息模块：

- 负责账号、日期、ID、栏目、星期等信息。
- 例：日期胶囊、底部信息条、星期轨道、角标。

内容区模块：

- 负责底图可写内容的区域。
- 例：大白板、便签纸、报纸栏、漫画格、HUD 面板。

特效模块：

- 负责增强视觉记忆点。
- 例：速度线、贴纸、胶带、光束、扫描线、纸张折角。

## 模块必须有参数

每个模块至少要说明：

- 可编辑文本
- 位置
- 尺寸
- 颜色
- 透明度
- 层级
- 是否可以删除
- 适合哪些模板

## 模块命名

模块 ID 使用英文和连字符，便于脚本读取。

模块中文名必须写清楚，便于用户理解。

示例：

```json
{
  "id": "comic-halftone-background",
  "中文名": "漫画半调网点背景",
  "类型": "背景模块",
  "状态": "候选"
}
```

## 模块合格标准

- 不能影响标题可读性。
- 不能遮挡账号、日期、ID 等基础信息。
- 不能让底图变成难以写字的花背景。
- 必须能被导出成固定尺寸 PNG。
- 必须能被记录进参数 JSON。
