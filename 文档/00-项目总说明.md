# 00-项目总说明

本项目的目标是把“封面 + 底图 + 图文日报”做成可复用的模块化设计系统。

它不是单个模板，也不是一个网页展示页。它应该成为 agent 可以稳定调用的设计生产工具。

## 核心理念

用搭积木的方式做封面和底图：

```text
画布 = 固定尺寸 + 安全边距 + 导出规则
模块 = 背景 / 标题 / 信息条 / 角色 / 内容区 / 特效
模板 = 多个模块组合出来的一套可用配方
产物 = HTML 渲染源 + PNG 图片 + 参数记录 + 验证报告
```

## 当前范围

第一阶段只做标准化骨架：

- 中文文档规则
- 公开素材候选表
- 素材授权台账规则
- 模块注册表
- 模板配方注册表
- agent 使用流程
- 验收标准

第一阶段暂不批量下载素材，避免版权和仓库体积失控。

## 公开仓库和私有仓库的边界

公开仓库：

- 放通用模块规则。
- 放公开安全素材。
- 放公开安全示例输出。
- 不绑定具体账号和具体 IP。

私有仓库：

- 放真实账号信息。
- 放私有角色图和产品图。
- 放具体运营账号的默认模板选择规则。
- 放真实发布项目的配置。

## 文件命名原则

- 面向人阅读的文档优先用中文文件名。
- Skill 生态要求的文件名保留英文，例如 `SKILL.md`、`README.md`、`scripts/`。
- 数据文件可以使用中文名，但字段含义必须清楚。

## 成功标准

一个合格的版本必须做到：

- agent 不需要从零设计。
- 用户能看懂规则。
- 素材来源能追溯。
- 模块和模板能复用。
- PNG 输出尺寸稳定。
- 每次生成都有验证结果。
