# 01-素材采集标准

素材采集的目标不是“越多越好”，而是找到能被模块化复用、授权清楚、视觉质量足够的素材。

## 优先采集类型

杂志类：

- 纸张纹理
- 分栏线
- 贴纸
- 条形码
- 胶带
- 印刷颗粒
- 价格标签

漫画类：

- 半调网点
- 爆炸框
- 对白框
- 速度线
- 分格线
- 拟声词贴纸

游戏类：

- RPG 图标
- 道具框
- 徽章
- 任务卡
- 像素边框
- 血条 / 能量条

星际科技类：

- HUD（抬头显示）框
- 雷达环
- 扫描线
- 星图
- 坐标网格
- 发光粒子

## 优先授权

优先级从高到低：

1. `CC0` / Public Domain（公有领域）
2. MIT / ISC / Apache-2.0 等开源许可
3. `CC BY`，可用但必须署名
4. 其他许可，必须逐条人工判断

不建议使用：

- 授权不明素材
- 只允许个人使用的素材
- 禁止商用的素材
- 明显来自知名 IP 的角色、徽标、截图
- 需要登录下载但没有明确授权说明的素材

## 候选来源

当前可先从这些来源找候选：

| 来源 | 适合类型 | 初步授权判断 | 备注 |
|---|---|---|---|
| Kenney | 游戏素材、UI、图标 | CC0 | 适合公开演示素材 |
| Poly Haven | 纹理、背景、材质 | CC0 | 适合纸张、金属、墙面等纹理 |
| Game-icons.net | 游戏图标 | CC BY 3.0 | 必须署名 |
| NASA | 星际、宇宙图像 | 多数内容可用但有限制 | 不能暗示 NASA 背书，不能乱用标志 |
| OpenGameArt | 游戏素材 | 混合授权 | 必须逐项核验 |
| Openverse | 素材搜索 | 混合授权 | 只是搜索入口，必须核原始页面 |

## 采集前检查

每个素材进入候选表前，至少记录：

- 原始来源链接
- 授权类型
- 是否需要署名
- 作者 / 机构名称
- 适合做哪个模块
- 是否适合公开仓库
- 版权风险等级

## 入库原则

候选素材不等于入库素材。只有同时满足下面条件才可以入库：

- 授权明确。
- 来源可追溯。
- 风格对模板有帮助。
- 文件命名清楚。
- 已写入 `数据/已入库素材清单.json`。
