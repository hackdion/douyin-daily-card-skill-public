# 04-模板配方规则

模板配方用来描述一套封面/底图由哪些模块组成。

## 配方结构

```text
模板配方 = 画布 + 背景模块 + 标题模块 + 信息模块 + 内容区模块 + 角色模块 + 特效模块
```

## 状态分级

`stable`：

- 可以被 agent 自动选择。
- 已经有示例输出。
- 已通过验证。

`beta`：

- 只能手动指定。
- 不允许自动默认使用。
- 适合继续试风格。

`draft`：

- 只是想法或结构草案。
- 不应生成正式产物。

## 通用模板选择原则

公开仓库只放通用选择原则，不绑定具体账号。

优先级：

```text
用户明确指定 > 内容用途 > 目标平台 > 可用素材 > 默认稳定模板
```

示例：

- 用户说“做漫画风”：优先选择漫画类模板。
- 用户说“做封面和底图”：优先选择同时支持 cover/base 的模板。
- 用户没有指定风格：选择 `stable` 且最稳的通用模板。

## 私有规则边界

某个账号自己的规则不写进公开配方。

例如“某个游戏账号周一默认用蓝图，周五默认用警报”，应该放到私有配置里，而不是公开仓库的硬规则。

## 配方必须记录的内容

- 模板 ID
- 中文名
- 状态
- 支持输出：封面 / 底图 / 正文页
- 适合用途
- 不适合用途
- 所需模块
- 所需素材
- 默认参数
- 示例输入
- 示例输出
- 验证结果
