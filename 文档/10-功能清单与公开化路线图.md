# 10-功能清单与公开化路线图

本文件用于回答一个核心问题：当前项目里有哪些能力已经存在，哪些已经进入公开仓库，哪些仍在私有工作区或实验阶段。

| 模块注册表 | 已接入 | `数据/模块注册表.json` | 记录背景、标题、信息条、角色等模块 |

## 需要补齐的能力

| 能力 | 优先级 | 原因 |
|---|---|---|
| 公开 demo 素材包 | 高 | 公开仓不能只有文字规则，必须有可视参考 |
| 素材候选预览页截图和示例 | 高 | 方便人工挑选，降低误入库风险 |
| 积木式编辑器增强 | 高 | 现有 beta 支持文字和滑杆，后续要补拖拽、模块删减、PNG 导出 |
| 模板配方注册表深度接入 | 高 | 当前已能从编辑器 JSON 生成 PNG，后续要让注册表直接驱动多模板批量渲染 |
| 私有素材覆盖层说明 | 中 | 让用户把自己的素材接入，但不污染公开仓 |
| MCP 服务包装 | 低 | 等 CLI 稳定后再做，不提前锁死接口 |

## 公开化原则

- 公开仓库放通用能力。
- 私有素材、真实账号、具体 IP 风格规则不放公开仓。
- 公开 demo 必须能运行，不能只有文字说明。
- 所有公开素材必须能追溯来源和授权。
- `draft` 和 `beta` 模板不能自动默认调用。

## 推荐落地顺序

1. 完成素材候选预览页。
2. 人工挑选第一批公开素材。
3. 下载并登记公开 demo 素材。
4. 把本地交互式模板改造成公开安全版。
5. 用 `scripts/validate-public-tools.mjs` 验收公开工具页。
6. 增强积木式编辑器 v1：拖拽、模块开关、批量输出。
7. 把模板配方注册表深度接到渲染脚本。
8. 再考虑 MCP 服务。
